# Tourism-Intelligence-FYP
Repository for my Final Year Project on *Volume to Value: Behavioural Segmentation & Forecasting of Sri Lankan Tourism Markets*.

## Project Overview
This project challenges the traditional, volume-centric approach to macroeconomic tourism forecasting. It applies an end-to-end analytical pipeline to mathematically segment the inbound tourism market of Sri Lanka and deploy targeted predictive models. 

Moving beyond simple headcounts, this project utilizes 'Total Bed Nights' as a proxy for economic value. By combining unsupervised machine learning (K-Means) with a competitive time-series forecasting tournament (SARIMA, Holt-Winters, Prophet), the pipeline identifies strategic market personas and generates a highly accurate 24-month forecast. The final output is an interactive Power BI dashboard designed to aid policymakers in targeted marketing and capacity planning.

## Files in this Repository
* **`Segmented Tourism Forecasting Code.ipynb`**: The main Google Colab Notebook containing the end-to-end analytical pipeline (EDA, Clustering, and Forecasting).
* **`Scraper.ipynb`**: Lightweight extraction script used to transform unstructured PDF tables into Excel sheets.
* **`SL_Tourism_Data.xlsx`**: The raw historical dataset compiled and validated from SLTDA reports.
* **`/Exported_Data/`**: The relational CSV flat files generated by the Python pipeline to feed the BI Dashboard (includes `Geography Master.csv`, `Historical Master.csv`, `Forecast Master.csv`, `Strategic Summary.csv`, and `Champion Markets.csv`).
* **`Tourism_Dashboard.pbix`**: The final interactive Power BI dashboard file.

## How to Use the Dashboard
The `Tourism_Dashboard.pbix` file works out of the box using Power BI's cached Import Mode. You can download it and immediately interact with the visualizations. 

If you wish to refresh the underlying data or run the Python pipeline yourself:
1. Download the `/Exported_Data/` folder to your local machine.
2. Open the `.pbix` file in Power BI Desktop.
3. Go to **Transform Data > Data Source Settings**.
4. Click **Change Source** and point it to the local folder where you saved the CSVs.

## Methodology
The project follows the CRISP-DM framework through the following phases:

1. **Data Ingestion & Preparation**
   - Automated PDF scraping and manual validation.
   - Outlier mitigation (IQR capping) and isolating the COVID-19 structural break.
2. **Feature Engineering**
   - Engineering behavioural metrics (Seasonality Index, Average Growth, Commitment Ratios).
3. **Exploratory Data Analysis (EDA)**
   - BCG Matrixing, Correlation checks, and Longitudinal Stability analysis.
4. **Market Segmentation (Clustering)**
   - Finding the optimal *k* (Silhouette, Elbow, CH, and DB indices).
   - K-Means algorithm execution and Principal Component Analysis (PCA) visualization.
   - ANOVA Feature Importance evaluation to map strategic clusters.
5. **Predictive Modelling (The Forecasting Tournament)**
   - Establishing an aggregate "Straw Man" baseline.
   - Executing competitive forecasting across clusters (SARIMA vs. Prophet vs. Holt-Winters).
   - Residual Diagnostics (ACF) to validate white noise.
   - Generating 24-month horizon forecasts with 95% risk bounds.
6. **Business Intelligence Deployment**
   - Exporting relational schemas for a Power BI Star Schema.
